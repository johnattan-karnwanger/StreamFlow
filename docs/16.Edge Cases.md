EC-01: Ввод кода в момент истечения срока
Триггер: Пользователь вводит код ровно в момент истечения 5 минут
Ожидаемое поведение:
1. Система считает код недействительным
2. Показывает ошибку "Код недействителен"
3. Предлагает запросить новый код
Риск: Если граница не обработана точно, пользователь может войти с истёкшим кодом — дыра в аутентификации
Связано: BR-01, UC-01, UC-02

---
EC-02: Повторный запрос кода до истечения срока старого
Триггер: Запрос отправлен напрямую через API, минуя фронтенд
Ожидаемое поведение:
1. Система не отправляет новый код
2. Возвращает ошибку "Код уже отправлен, подождите"
Риск: Без проверки на бэкенде возможен спам запросов к SMS-шлюзу
Связано: UC-01, UC-02

EC-03: Мут на срок менее 1 минуты через API

Связано: BR-02, UC-07

EC-14: Обрыв RTMP-соединения во время стрима
Триггер: OBS теряет соединение с медиасервером во время активной трансляции
Ожидаемое поведение:
1. Система переводит стрим в статус Reconnecting
2. Уведомляет стримера "Соединение потеряно, ожидание восстановления"
3. Ожидает восстановления соединения в течение 5 минут
4. Если соединение не восстановлено — завершает стрим и сохраняет VOD
Риск: Без статуса Reconnecting стрим завершится мгновенно при любом кратковременном сбое сети
Связано: NFR-Reliability-1, FR-24

EC-15: Ручная остановка стрима в статусе Reconnecting
Триггер: Стример нажимает "Остановить стрим" пока стрим в статусе Reconnecting
Ожидаемое поведение:
1. Система отменяет ожидание переподключения
2. Переводит стрим в статус Ended
3. Сохраняет VOD
Риск: Без обработки этого сценария система может продолжать ждать переподключения даже после ручной остановки
Связано: FR-20, FR-24

EC-20: Глобальный бан пользователя с активной подпиской
