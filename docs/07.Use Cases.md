UC-01: Регистрация
Актор: Гость  
Цель: Создать аккаунт в системе
Precondition: Гость на сайте 

Основной поток:
1. Гость нажимает кнопку "Регистрация"
2. Система открывает окно регистрации со строками "Номер телефона", "Пароль" и "Подтвердить пароль"
3. Гость вводит номер телефона, пароль и подтверждает пароль
4. Система отправляет код на номер телефона
5. Система открывает строку с возможностью ввода кода и пишет "Вам отправлен код на телефон, введите его снизу"
6. Гость вводит код
7. Система создаёт аккаунт и автоматически выполняет вход

Альтернативные потоки:
- 3А. Невалидный номер телефона → система показывает ошибку: "Номер телефона введен неверно, пожалуйста введите допустимый номер телефона"
- 3Б. Невалидный пароль → система показывает ошибку: "Пароль может содержать только буквы, цифры и спец. знаки"
- 3В. Подтверждение пароля не совпадает с паролем → система показывает ошибку: "Подтвержденный пароль не совпадает с паролем"
- 3Г. Номер телефона уже зарегистрирован → система показывает ошибку: "Этот номер телефона уже зарегистрирован в системе, пожалуйста войдите в систему или введите другой номер" 
- 4А. SMS-шлюз недоступен или возвращает ошибку → система показывает ошибку: "Сервис отправки СМС временно недоступен, пожалуйста, попробуйте позже". Процесс останавливается, пользователь может запросить код повторно.
- 6А. Неверный код → система показывает ошибку: "Неверный код, попробуйте снова". Гость может запросить новый код.
- 6Б. Срок кода истек → система показывает ошибку: "Код недействителен". Пользователь может запросить новый код.

Postcondition: Гость зарегистрирован и превращается в роль "Зритель"

UC-02: Вход в систему
Актор: Пользователь  
Цель: Войти в аккаунт в системе
Precondition: Пользователь на сайте

Основной поток:
1. Пользователь нажимает кнопку "Вход"
2. Система открывает окно входа со строками "Номер телефона" и "Пароль"
3. Пользователь вводит номер телефона и пароль
4. Система отправляет код на номер телефона
5. Система открывает строку с возможностью ввода кода и пишет "Вам отправлен код на телефон, впишите его снизу"
6. Пользователь вводит код
7. Система выполняет вход

Альтернативные потоки:
- 3А. Неверный номер телефона или пароль → система показывает ошибку: "Неверный пароль или номер телефона"
- 6А. Неверный код → система показывает ошибку: "Неверный код, попробуйте снова". Пользователь может запросить новый код.
- 6Б. Срок кода истек → система показывает ошибку: "Код недействителен". Пользователь может запросить новый код.

Postcondition: Пользовать вошел в систему

UC-03: Поиск и заход на определенный стрим
Актор: Пользователь
Цель: Зайти на определенный стрим
Precondition: Пользователь находится на сайте

Основной поток:
1. Пользователь нажимает строку "Поиск" 
2. Система открывает окно поиска
3. Пользователь вводит определенное название стрима и нажимает Enter
4. Система открывает окно со списком найденных стримов
5. Пользователь нажимает на изображение или название стрима
6. Система переносит пользователя на вкладку с этим стримом

Альтернативные потоки:
- 4А. Список пустой → система показывает надпись: "Ничего не найдено"
- 6А. Стрим закончен → система показывает надпись: "Данный стрим завершился". Система переносит пользователя во вкладку с поиском.

Postcondition: Пользователь находится на стриме, либо возвращён на страницу поиска.

UC-04: Написать сообщение в чате
Актор: Зритель
Цель: Написать сообщение в чат
Precondition: Зритель находится на стриме

Основной поток:
1. Зритель нажимает на строку снизу чата "Написать"
2. Зритель набирает текст и нажимает Enter
3. Система отправляет сообщение в чат

Альтернативные потоки:
- 3А. Зритель находится в муте → система показывает ошибку: "Вы замьючены в данном чате. Мут истекает через N (пример: 03:05:47)"
- 3Б. Стрим завершился пока зритель писал → кнопка отправки блокируется

Postcondition: Зритель написал сообщение в чат, либо показывается ошибка мута

UC-05: Оформление подписки
Актор: Зритель
Цель: Оформить подписку на канал
Precondition: Зритель авторизован и находится во вкладке канала

Основной поток:
1. Зритель нажимает кнопку "Подписку"
2. Система открывает окно с возможностями подписки, и кнопкой "Оформить подписку за N сумму"
3. Зритель нажимает "Оформить подписку за N сумму"
4. Система открывает окно привязки карты
5. Зритель вводит свои данные и нажимает "Оплатить"
6. Система обращается в платежную систему по АПИ
7. Система получает успешный статус оплаты от платежной системы
8. Система создаёт запись в таблице Subscription, Transaction и сообщает пользователю об успешной оформлении подписки.

Альтернативные потоки:
- 5А. Зритель ввел невалидные символы → система выдает ошибку: "Данные о карте должны содержать только цифры"
- 7А. Недостаточно средств → система выдает ошибку: "На счете недостаточно средств", затем возвращает зрителя в окон привязки карты
- 7Б. Платёжная система недоступна → система выдает ошибку: "Платежная система не отвечает", затем возвращает зрителя в окон привязки карты

Postcondition: Зритель оформил подписку

UC-06: Настройка и запуск стрима
Актор: Стример
Цель: Настроить и запустить стрим
Precondition: Стример находиться во вкладке канала

Основной поток:
1. Стример нажимает кнопку "Настроить и запустить стрим"
2. Система отображает окно настройки
3. Стример настраивает название, хэштеги, тип стрима и обложку стрима
4. Стример нажимает "Показать stream key"
5. Система отображает stream key
6. Система отображает статус "Ожидание подключения стримингового ПО..."
7. Стриминговое ПО подключается по RTMP
8. Система переводит канал в статус live и уведомляет: "Ваш стрим запущен"

Альтернативные потоки:

- 6А. Соединение не установлено в течение 30 секунд → система показывает: "Подключение не установлено. Проверьте stream key в стриминговом ПО"

Postcondition: Стрим запущен

UC-07: Выдача мута
Актор: Модератор канала
Цель: Выдать зрителю мут на стриме
Precondition: Модератор канала находится на стриме

Основной поток:
1. Модератор нажимает на зрителя в чате
2. Система открывает окно взаимодействия
3. Модератор нажимает "Выдать мут"
4. Система открывает окно выдачи мута с таймером
5. Модератор выставляет время и нажимает "Выполнить"
6. Система отправляет зрителя в мут на выбранное время, и создаёт запись в таблице Moderation

Альтернативные потоки:

- 3А. Модератор канала выбрал другого модератора этого же канала или стримера → система показывает: "К данному зрителю нельзя применить мут"
- 5А. Модератор канала выбрал срок менее 1 минуты → система показывает: "Выберите 1 минуту и больше"

Postcondition: Зритель замьючен

---

UC-08: Выдача глобального бана пользователю
Актор: Модератор контента
Цель: Выдать пользователю системы глобальный бан
Precondition: Модератор контента авторизован в системе

Основной поток:
1. Модератор контента нажимает "Панель модерации"
2. Система открывает панель модерации
3. Модератор контента открывает жалобу на определенного зрителя
4. Система открывает данные данного зрителя
5. Модератор контента нажимает "Забанить"
6. Система открывает окно выдачи бана с таймером
7. Модератор контента выставляет время и нажимает "Выполнить"
8. Система отправляет пользователя в бан на выбранное время, и создаёт запись в таблице Moderation

Альтернативные потоки:

- 3А. Жалоб нет → система показывает пустой список. Модератор может создать жалобу вручную
- 5А. Модератор контента выбрал другого модератора контента → система показывает: "К данному пользователю нельзя применить бан"
- 7А. Модератор контента выбрал срок менее 24 часов → система показывает: "Выберите 24 часа и больше"

Postcondition: Пользователь забанен в системе
